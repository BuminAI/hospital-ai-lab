# 병원 AI 연구소 — 운영 규정

## 사이트 정보
- 이름: 병원 AI 연구소 (Hospital AI Lab)
- 주소: https://hospital-ai-lab.com
- 스택: Astro + GitHub Pages. 글은 src/content/blog/의 마크다운
- 사실 정보의 유일한 출처: briefing.md (여기 없는 이력·경력은 절대 창작 금지)

## 미션과 독자
- 미션: 의료 현장의 언어로 인공지능을 연구하고 나눈다
- 독자: 병원 행정·간호 등 병원 종사자 중심 (의료 AI 학습자 포함)
- 관점: 의사(임상의) 전용 관점으로 쓰지 않는다. 도입 결정·제안서 검토·예산은 행정의 일, 알림 대응·업무 흐름 변화는 간호 현장의 일이라는 눈높이를 우선한다

## 사실 검증 (절대 원칙 — 모든 규칙에 우선)
- 사이트에 올리는 모든 자료의 사실 주장에는 객관적 출처가 있어야 한다
- 할루시네이션(근거 없이 생성된 정보)은 절대 불가. 조금이라도 의심되면 그 내용은 싣지 않는다
- 발행 전 모든 출처 링크를 실제 접속해 (a) 살아 있는지 (b) 해당 주장을 실제로 뒷받침하는지 확인한다
- 출처를 확인하지 못한 문장은 삭제한다. 삭제할 수 없으면 [확인 필요]로 표시하고 오너 승인 전에는 발행하지 않는다
- 저자 관련 사실의 출처는 briefing.md 하나뿐이다

## 글 작성 규칙
- 한국어, 담백한 존댓말. 과장·감탄 남발 금지
- 길이 1,500~2,500자. 구조: 도입(왜 중요한가) → 소제목 2~4개 → 핵심 요약
- 파일명: src/content/blog/YYYY-MM-DD-슬러그.md
- frontmatter: title, description, pubDate, category(논문리뷰|AI도구|칼럼|소식), draft
- 근거가 필요한 주장에는 출처 링크. 출처를 못 찾으면 그 문장은 쓰지 않는다
- 원문을 길게 옮기지 않는다(저작권). 인용은 짧게, 나머지는 내 말로

## 의료 콘텐츠 규칙 (절대 준수)
- 특정 환자·개별 사례에 대한 진단·치료 조언 금지. 일반 정보만
- "완치", "100%", "부작용 없음" 등 효과 보장·과장 표현 금지
- 특정 의료기관·시술의 홍보성 내용 금지 (의료광고 심의 리스크 회피)
- 환자 식별 가능 정보 절대 금지
- 모든 글 하단에 면책 문구 포함 (briefing.md 참조)
- 확신 없는 의학 정보는 쓰지 말고 [확인 필요]로 표시 후 오너에게 보고

## 작업 규칙
- 새 글은 draft: true로 작성 → 브랜치 + PR 생성 → 오너가 머지해야 발행
- 고정 페이지의 소소한 수정은 main에 직접 커밋 가능. 단, 빌드 확인 후
- 모든 변경 전 npm run build 성공 확인
- 커밋 메시지는 한국어: "글: 제목" / "수정: 내용" / "유지보수: 내용"
- 금지: briefing.md에 없는 사실 창작, 개인정보 노출, node_modules 커밋, 오너 승인 없는 대규모 구조 변경·라이브러리 추가
